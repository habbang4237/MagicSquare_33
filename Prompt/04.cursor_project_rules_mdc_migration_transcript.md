## 3. `.mdc` 형식으로 바꿀 때 핵심

Cursor Project Rules는 **`.cursor/rules/*.mdc`** + 상단 YAML입니다.

```yaml
---
description: 한 줄 요약 (규칙 선택 UI에 표시)
alwaysApply: true   # 또는 false
globs: **/*.py      # alwaysApply가 false일 때만 의미 있음
---
```

### `alwaysApply` / `globs` 권장안

| 파일 | 권장 설정 | 이유 |
|------|-----------|------|
| `magicsquare-project.mdc` | `alwaysApply: true` | 이름·계약·문서·디렉터리는 항상 맥락 |
| `magicsquare-forbidden.mdc` | `alwaysApply: true` | 실수 방지용 “절대 금지” |
| `magicsquare-ecb-architecture.mdc` | `globs: src/magicsquare/**/*.py` (+ 필요 시 `tests/**/*.py`) | 레이어 작업할 때만 |
| `magicsquare-tdd-testing.mdc` | `globs: tests/**/*.py` (+ RED/GREEN 시 `src/**`도 열면 `**/*.py`) | 테스트·TDD 단계 중심 |
| `magicsquare-python-code-style.mdc` | `globs: **/*.py` | Python 파일 편집 시 |

`globs`는 여러 패턴이 필요하면 Cursor 버전에 따라 `globs`를 배열로 쓰거나, 파일을 둘로 나누는 방식을 쓰면 됩니다. **만들기 전에** “항상 vs 파일 열릴 때만”만 정해 두면 됩니다.

## 4. 내용 변환 시 원칙

1. **한 파일 = 한 관심사** — skill 권장처럼 파일당 대략 **50줄 전후·한 주제**가 이상적입니다. 지금 `.cursorrules`는 240줄이라 5분할이 적절합니다.  
2. **액션 가능한 문장** — `must_not:` 목록은 “하지 말 것 / 대신 할 것” 쌍으로 유지 (`forbidden`은 이미 잘 되어 있음).  
3. **권위 문서는 project에만** — `Report/01`, `Report/02`, `README` 링크는 `magicsquare-project.mdc` 한곳에만 두고, 나머지는 “project 규칙 참조” 한 줄로 줄입니다.  
4. **중복 정리**  
   - `entity → boundary import 금지`: architecture에 상세, forbidden에는 한 줄  
   - `RED 없이 prod 수정 금지`: tdd-testing에 상세, forbidden에는 한 줄 + `tdd_rules` 참조  
5. **예시는 해당 파일에만** — 예: `except SpecificError` 예시는 `forbidden` 또는 `python-code-style` 중 한 곳만.

## 5. `.cursorrules`와의 관계

- **레거시**: 루트 `.cursorrules`는 예전 방식, **`.cursor/rules/*.mdc`가 권장**입니다.  
- **이중 적용 주의**: 둘 다 두면 비슷한 내용이 두 번 들어가 토큰만 늘고, 가끔 충돌합니다.  
- **마이그레이션 후 선택**  
  - **삭제** (가장 깔끔)  
  - 또는 `.cursorrules`에 3~5줄만: “규칙은 `.cursor/rules/magicsquare-*.mdc`로 이전됨”
