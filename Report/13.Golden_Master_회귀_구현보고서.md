## 3. Golden Master 설계

### 3.1 Approve 패턴

```
pytest / generate script
        │
        ▼
 golden_master_expected.txt 존재?
        │
   ┌────┴────┐
  No        Yes
   │          │
   ▼          ▼
 현재 출력    actual vs expected
 자동 생성    byte-equal 비교
   │          │
   │     ┌────┴────┐
   │    일치      불일치
   ▼     ▼          ▼
 PASS  PASS    unified diff + FAIL
```

| 모드 | 명령 |
|------|------|
| pytest compare | `pytest -m golden_master -v` |
| pytest approve | `pytest -m golden_master --approve-golden-master -v` |
| script write | `python scripts/generate_golden_master.py` |
| script check (CI) | `python scripts/generate_golden_master.py --check` |

### 3.2 Test-Case ↔ 검증

| Test-ID | 시나리오 | Golden block | Contract 검증 |
|---------|----------|--------------|---------------|
| GM-TC-01 | attempt A (small-first) 성공 | `Output: int[6]` | row-major, 1-index, m1→slot1 |
| GM-TC-02 | attempt B (reverse) 성공 | `Output: int[6]` | A 실패·B 성공, m2→slot1 |
| GM-TC-03 | 빈칸 수 ≠ 2 | `Error: E_EMPTY_COUNT` | ErrorResponse code + message |
| GM-TC-04 | 비零 중복 | `Error: E_DUPLICATE` | ErrorResponse code + message |
| GM-TC-05 | A·B 모두 실패 | `Error: E_NO_SOLUTION` | ErrorResponse code + message |

### 3.3 ECB 진입점

- 캡처: `PuzzleBoundary.receive` → `SolvePuzzleUseCase` (**Mock 없음**)
- Entity 직접 호출 없음 (Boundary→Control 준수)
- contract_checks는 **검증 보조**로 Domain 서비스(`EmptyCellLocator`, `MagicSquareValidator` 등) 사용 — 테스트 레이어 허용

### 3.4 파서 주의

- 출력 리스트 `[2, 1, 5, 3, 3, 7]`이 블록 헤더로 오인되지 않도록 파서를 `^\[GM-TC-\d{2}\]$`로 제한
