Dual-Track UI + Logic TDD 코치 역할로, Report/09 설계표 중 **아직 pytest 함수가 없던 Test-ID**에 대해 **RED Skeleton**만 작성했다.

| 순서 | 사용자 요청 | 결과 |

| 1 | Track A/B RED Skeleton 작성 (`pytest.fail`만, prod 금지) | `tests/boundary/` 3파일, `tests/entity/` 4파일, `tests/conftest.py` |
| 2 | `pytest tests/boundary/ tests/entity/ -v` | 13 passed, 8 collection errors |
| 3 | Report 생성 + Prompt Transcript Export | 본 보고서, `Prompt/10.*` |

**세션 금지 사항 (준수):**

- `src/` 운영 코드·stub·GREEN·REFACTOR
- 기존 Full RED `test_input_contract_ac_fr_01_01.py` 수정
- `test_user.py` 수정
- 실제 기대값 assert, `skip`/`xfail`, Mock `call_count` assert

**고정 계약 (Skeleton 주석·docstring 기준):**

| 항목 | 내용 |
|------|------|
| Skeleton 본문 | `pytest.fail("RED: <Test-ID> — …")` 한 줄 |
