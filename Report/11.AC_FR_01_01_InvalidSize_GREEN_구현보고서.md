## 4. 구현 내용

### 4.1 ECB 의존 방향

```
boundary (PuzzleBoundary, InputContractValidator, ErrorResponse)
    → control (SolvePuzzleUseCase 스텁만 import)
    ✗ entity (미참조)
```

### 4.2 `InputContractValidator.validate`

- `grid is None` → `ErrorResponse(INVALID_SIZE_CODE, INVALID_SIZE_MESSAGE)` 반환
- 그 외 → `None` (추가 size 검증 없음)

### 4.3 `PuzzleBoundary.receive`

1. `validation_result = self._validator.validate(grid)`
2. `validation_result is not None` → 즉시 `ErrorResponse` 반환 (**`execute` 미호출**)
3. 그 외 → `self._use_case.execute(grid)` (본 테스트 경로에서 미실행)

### 4.4 `SolvePuzzleUseCase`

- `execute`, `resolve` 시그니처만 제공
- 본문 `raise NotImplementedError` — Domain 로직 없음
