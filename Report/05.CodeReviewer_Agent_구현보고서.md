## 3. 에이전트 설정

| frontmatter | 값 | 의미 |
|-------------|-----|------|
| `name` | `code-reviewer` | 서브에이전트 식별자 |
| `description` | 전문 코드 품질 검토자… proactively | 위임·자동 호출 트리거 문구 |
| `model` | `inherit` | 부모 대화와 동일 모델 사용 |
| `readonly` | `true` | 코드·파일 수정 없이 리뷰만 수행 |

**위치:** `.cursor/agents/` (프로젝트 레벨) — 팀 저장소에 커밋하면 공유 가능. 사용자 전역 `~/.cursor/agents/`보다 우선한다.

## 4. 리뷰 지침 구조

| 섹션 | 내용 |
|------|------|
| 호출 시 절차 | 범위 파악 → `.cursor/rules/`·Report·README 대조 → ECB 레이어별 검토 → 구조화 리포트 |
| 정확성·버그 | 입출력 계약, 마방진 불변조건, 경계값, 에러 매핑 |
| ECB·규칙 | 의존 방향, `magicsquare-forbidden` 금지 패턴, 레이어 책임 |
| TDD·테스트 | RED/GREEN, Mock 정책(Dual-Track), Test-ID, AAA |
| Python 스타일 | PEP8, type hints, docstring |
| 성능 | 4×4·백트래킹 맥락의 제안 수준 (과도한 최적화 지양) |
| 출력 형식 | 요약 → Critical → Warning → Suggestions → 규칙 위반 태그 |
| 행동 원칙 | 근거 인용, 수정 금지, 범위 내 최소 수정 권고 |

## 5. Project Rules·Report 연동

리뷰어가 참조하도록 본문에 명시한 권위 소스:

| 유형 | 경로 |
|------|------|
| Project Rules | `magicsquare-project`, `magicsquare-forbidden`, `magicsquare-ecb-architecture`, `magicsquare-tdd-testing`, `magicsquare-python-code-style` |
| 설계·계약 | `Report/01`, `Report/02`, `README.md` |

**규칙 위반 태그** (프로젝트 AI 작업 순서와 동일):

- `TDD WARNING:` — TDD·테스트 무결성 위반
- `ECB LAYER WARNING:` — 레이어·의존 방향 위반
- `COVERAGE WARNING:` — 커버리지 기준 미달

## 6. 사용 방법

채팅 또는 Task 위임 예:

```text
code-reviewer 서브에이전트로 src/entity/ 변경분 리뷰해 줘
```

```text
@entity /code-reviewer 리뷰
```

`readonly: true`이므로 리뷰 결과는 **피드백만** 반환하며, 수정은 별도 에이전트·본 대화에서 수행한다.
