## 4. User 엔티티 (ECB Entity)

### 4.1 위치·책임

- **레이어:** Entity (`src/entity/`)
- **역할:** 퍼즐 학습자·계정 도메인 표현 (Report/02의 Grid·Solver와 **독립**된 보조 엔티티)
- **의존:** boundary/control import **없음** (ECB 준수)

### 4.2 불변조건

| ID | 규칙 | 실패 코드 |
|----|------|-----------|
| U-1 | `user_id` strip 후 길이 1~64 | `INVALID_USER_ID` |
| U-2 | `display_name` strip 후 길이 1~50 | `INVALID_DISPLAY_NAME` |
| U-3 | `email` 길이 3~254, `local@domain.tld` 형식 | `INVALID_EMAIL` |
| U-4 | 생성 시 `is_active` 기본 `True` | — |
| U-5 | frozen — 필드 직접 변경 불가 | `AttributeError` |

### 4.3 공개 API

```text
User.create(user_id, display_name, email, *, is_active=True) -> User
User.deactivate() -> User   # is_active=False 복사본
```

### 4.4 테스트 매핑 (U-ENT-xx)

| Test-ID | 시나리오 | 기대 |
|---------|----------|------|
| U-ENT-01 | 유효 입력 + 공백·대소문자 정규화 | 정규화된 User |
| — | 비활성 생성 | `is_active=False` |
| — | 빈/과장 user_id | `INVALID_USER_ID` |
| — | 빈/과장 display_name | `INVALID_DISPLAY_NAME` |
| — | 잘못된 email (parametrize 4건) | `INVALID_EMAIL` |
| — | immutable / deactivate / equality | 동작 검증 |
